## 📋 User Story

**Como** administrador do sistema,
**Quero** configurar as credenciais SMTP através de uma interface web e realizar envios de teste,
**Para** que eu possa validar a conexão antes de disparar campanhas em massa e evitar falhas por configurações incorretas no `.env`.

---

## 🎯 Objective

Implementar uma página de configurações no Frontend e uma API de gerenciamento de configurações no Backend para persistir as credenciais SMTP no banco de dados (PostgreSQL), substituindo a dependência exclusiva do arquivo `.env` para o envio de e-mails.

## 🎯 Acceptance Criteria

```gherkin
GIVEN que sou um administrador logado na aplicação
WHEN eu acesso a página de "Configurações"
THEN devo ver um formulário com os campos de SMTP (Host, Port, Usuário, Senha, Remetente) ✅

GIVEN que preenchi os campos de SMTP
WHEN eu clico em "Testar Conexão" e informo um e-mail de destino
THEN o sistema deve tentar enviar um e-mail de teste e retornar "Sucesso" ou o erro detalhado do servidor SMTP ✅

GIVEN que as configurações foram salvas com sucesso
WHEN eu realizar um novo disparo em massa
THEN o sistema deve utilizar as credenciais salvas no banco de dados em vez das variáveis do .env ✅
```
