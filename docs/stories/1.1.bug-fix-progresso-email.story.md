# Story 1.1: Acompanhamento de progresso de envio de email não está funcionando

## 📋 Status
**Status**: [Done]
**Executor**: @dev
**Quality Gate**: @qa

## 📖 Story
**As a** Usuário,
**I want** ver atualizações em tempo real do progresso de envio após realizar o envio do meu formulário,
**so that** eu consiga acompanhar o progresso sem precisar dar refresh na página e evitar loops de requisições.

## ✅ Acceptance Criteria
1. [x] O progresso (barra de progresso) deve avançar conforme os emails são enviados.
2. [x] Os componentes da página (listagem de emails, assunto, corpo) devem ser exibidos e atualizados conforme o envio.
3. [x] A funcionalidade não deve exigir refresh da página para mostrar o progresso.
4. [x] O refresh da página não deve causar um loop de requisições redundantes.
5. [x] O endpoint `/status/{jobId}` deve ser consumido corretamente para refletir o estado do job.
