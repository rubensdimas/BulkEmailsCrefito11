## 📋 User Story

**Como** desenvolvedor,
**Quero** que as migrações do banco de dados sejam executadas automaticamente ao subir o ambiente,
**Para** que eu não precise rodar comandos manuais e o sistema esteja pronto para uso imediatamente após o docker compose up.

---

## 🎯 Objective

Garantir que as tabelas do banco de dados sejam criadas corretamente durante o processo de inicialização do container backend. Atualmente, o comando de migração no docker-compose parece não estar surtindo efeito ou falhando silenciosamente, deixando o banco vazio e o sistema inoperante.

## 🎯 Acceptance Criteria

```gherkin
GIVEN um banco de dados vazio e os containers desligados
WHEN o comando "docker compose up -d" é executado
THEN o container backend deve executar "npm run migrate" com sucesso antes de iniciar o servidor
AND todas as tabelas definidas em backend/src/migrations devem estar presentes no banco "bulkmail"

GIVEN o ambiente já está rodando
WHEN novas migrações são adicionadas e o container é reiniciado
THEN as novas migrações devem ser aplicadas automaticamente sem perda de dados existentes
```

## 📝 Dev Notes

### Key Files
- `docker-compose.yml`: Atualmente usa `sh -c "npm run migrate && npm run dev"`
- `backend/package.json`: Script `migrate` definido como `knex migrate:latest`
- `backend/knexfile.ts`: Configuração do Knex para o banco Postgres (aponta para `./src/migrations`)
- `backend/src/migrations/001_initial_schema.sql`: Schema atual que precisa de um wrapper Knex
- `backend/src/migrations/20260428000000_initial_schema.ts`: Wrapper Knex criado
- `README.md`: Documentação do deploy

### Technical Notes
- **Knex SQL Issue**: O Knex não executa arquivos `.sql` diretamente na pasta de migrations. É necessário criar um arquivo `.ts` que use `knex.schema.raw()` para carregar o SQL.
- **Path Correction**: O caminho correto das migrações é `backend/src/migrations`, não `backend/src/database/migrations`.
- **Wait-for-DB**: Adicionado loop `until pg_isready` no `docker-compose.yml` e instalado `postgresql-client` no Dockerfile do backend.

### Testing
- **Test command**: `docker compose exec postgres psql -U bulkmail -d bulkmail -c "\dt"`
- **Verification**: O output deve listar as tabelas `jobs` e `email_logs`.
- **Standards**: As migrações devem ser idempotentes e reversíveis (implementar `down` no wrapper).
- **Frameworks**: Knex CLI para inspeção e `psql` para validação direta.
