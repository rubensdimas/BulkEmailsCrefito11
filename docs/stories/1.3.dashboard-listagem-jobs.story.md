# Story 1.3: Dashboard com listagem de jobs

## 📋 Status
**Status**: [Done]
**Executor**: @dev
**Quality Gate**: @qa

## 📖 Story
**As a** Usuário,
**I want** ver e acompanhar uma listagem de todos os jobs de envio de e-mail,
**so that** eu consiga gerenciar o histórico de envios e acompanhar o progresso de múltiplos jobs sem precisar saber o ID exato de cada um.

## ✅ Acceptance Criteria
1. [x] Deve haver um novo link "Dashboard" no Navbar do frontend.
2. [x] A listagem de jobs deve exibir: Data de criação, Total de e-mails e Progresso (%) atual.
3. [x] A listagem deve suportar paginação de 20 itens (tanto no frontend quanto no backend).
4. [x] Deve ser possível filtrar os jobs (Query Builder no backend e campos de filtro no frontend).
5. [x] Deve ser possível excluir um job da lista (e do banco de dados), mas apenas se o status do job for "concluído".
6. [x] Se não houver jobs, uma tela de "Empty State" deve ser exibida com um link para criar um novo envio.
7. [x] A visualização deve incluir um "Skeleton Screen" durante o carregamento inicial dos dados.
