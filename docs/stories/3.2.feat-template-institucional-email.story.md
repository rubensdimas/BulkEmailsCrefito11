## 📋 User Story

**Como** administrador do CREFITO11,
**Quero** que os e-mails enviados pelo sistema tenham header e rodape institucionais com a marca do CREFITO11,
**Para** que as mensagens deixem de parecer texto simples e passem credibilidade visual alinhada ao manual da marca.

---

## 🎯 Objective

Criar um template HTML institucional reutilizavel para todos os e-mails enviados pelo BulkMail Pro, envolvendo o conteudo criado no Quill com header, logo, area de corpo e rodape alinhados ao manual da marca em `assets/CREFITO 11 - Manual da Marca.pdf` e usando arquivos de logo de `assets/Logos crefito11 png`.

O template deve preservar o conteudo HTML atual do usuario, manter fallback em texto puro quando possivel e funcionar em clientes de e-mail comuns com CSS inline/table layout.

## 🎯 Acceptance Criteria

```gherkin
GIVEN um usuario compoe um e-mail com conteudo HTML no editor Quill
WHEN o sistema processa o envio
THEN o HTML enviado pelo Nodemailer deve conter um header institucional do CREFITO11
AND deve preservar integralmente o conteudo criado pelo usuario no corpo da mensagem
AND deve conter um rodape institucional

GIVEN o template usa a logo do CREFITO11
WHEN o destinatario abre o e-mail
THEN a logo deve renderizar por CID inline ou URL absoluta publica
AND o HTML nao deve referenciar caminhos locais como "assets/..."

GIVEN o corpo do e-mail possui variaveis como "{{nome}}"
WHEN o job e processado com variables.nome preenchido
THEN a mensagem final deve substituir a variavel no conteudo enviado
AND nao deve quebrar o header ou o rodape do template

GIVEN o payload de envio nao traz campo text
WHEN o e-mail e enviado
THEN o sistema deve gerar fallback em texto puro a partir do conteudo final
AND o envio deve continuar aceito por clientes que bloqueiam HTML
```

## 📝 Dev Notes

### Key Files

- `backend/src/queue/processors.ts`: ponto atual que chama `transporter.sendMail`
- `backend/src/queue/emailQueue.ts`: tipos de job de e-mail
- `backend/src/controllers/sendController.ts`: recebe `html`, `text`, `templateId` e `variables`
- `backend/src/services/idempotencyService.ts`: ja contem helpers de variaveis de template e sanitizacao basica de HTML
- `assets/CREFITO 11 - Manual da Marca.pdf`: fonte para tokens visuais oficiais
- `assets/Logos crefito11 png/CREFITO 11 - Marca - Pos Completa.png`: candidato principal para header em fundo claro

### Technical Notes

- O renderer deve ser centralizado no backend para garantir que todos os envios, incluindo campanhas em fila, recebam o mesmo template.
- Para compatibilidade com e-mail, preferir HTML simples com tabelas, largura maxima em torno de 600px e CSS inline.
- Se usar CID inline, o `sendMail` deve receber `attachments: [{ filename, path, cid }]` e o HTML deve referenciar `src="cid:<cid>"`.
- Nao inserir scripts, CSS externo ou dependencia de fontes remotas no e-mail.
- O template deve continuar permitindo conteudo vindo do Quill, mas qualquer sanitizacao deve respeitar a story 1.5 e nao remover formatacao basica esperada.

### Testing

- Unit: renderer retorna HTML com header, body e footer.
- Unit: renderer nao referencia `assets/...` diretamente no atributo `src`.
- Unit: substituicao de variaveis continua funcionando.
- Unit/integration: `sendEmail` envia payload para Nodemailer com `html`, `text` e `attachments` quando aplicavel.
- Manual: envio de teste para conta real validando logo, layout e fallback.
