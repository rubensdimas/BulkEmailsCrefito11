# Story 1.2: Alteração de campos formulário inicial frontend

## 📋 Status
**Status**: [Approved]
**Executor**: @dev
**Quality Gate**: @qa

## 📖 Story
**As a** Usuário,
**I want** que os campos de assunto e corpo do e-mail tenham cores normais quando habilitados,
**so that** eu consiga ler e preencher as informações com clareza sem confusão visual de "dark theme" não solicitado.

## ✅ Acceptance Criteria
1. [ ] Ao carregar um arquivo .xlsx no primeiro campo, os campos "Assunto" e "Corpo do E-mail" devem ser liberados (disabled: false).
2. [ ] Quando os campos forem liberados, a cor de fundo e do texto deve ser a padrão do formulário (normal light theme/layout padrão), removendo o aspecto de "cinza escuro".
3. [ ] A funcionalidade de preenchimento deve estar disponível imediatamente após a habilitação.
4. [ ] O comportamento visual deve ser consistente entre diferentes navegadores.

## 🗒️ Dev Notes
- **Tecnologias**: React + TailwindCSS.
- **Contexto**: O problema ocorre especificamente na transição de estado após o processamento do arquivo de entrada.
- **Arquivos Relevantes**:
  - `frontend/src/components/EmailForm/EmailForm.tsx`
  - `frontend/src/index.css`
- **Hipótese**: A regra `:root { color-scheme: light dark; }` no `index.css` está permitindo que o navegador aplique cores escuras nativas aos campos habilitados se o SO estiver em Dark Mode, enquanto o componente só define explicitamente o fundo para o estado `disabled` (`bg-gray-100`).
- **Dica**: Forçar `bg-white` nos inputs/textareas do `EmailForm.tsx` ou alterar o `color-scheme` para `light` no `index.css`. Utilizar o **Playwright MCP** para facilitar a execução dos testes E2E sem dependências pesadas no ambiente de dev inicial.

### Testing
- **Padrões**: Playwright para testes E2E (Verificação de atributos CSS e estados).
- **Cenários**:
  - SO em Light Mode -> Campo habilitado deve ser branco.
  - SO em Dark Mode -> Campo habilitado deve ser branco (não cinza escuro).
  - Campo desabilitado -> Deve ser cinza claro (`bg-gray-100`).
