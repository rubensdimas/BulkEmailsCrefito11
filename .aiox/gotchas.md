### Job Status column vs Real-time computation divergence

**[WARNING]**

The "status" column in the jobs table is a static snapshot and can become stale during active processing.

**Workaround:** Always use JobStatusService.computeJobStatus() for real-time accuracy. Use the sync logic (shouldSyncStatus) to update the DB column upon completion to maintain listing performance.
